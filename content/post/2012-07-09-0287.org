#+title: 新USBメモリを安全に使う方法
#+date: 2012-07-09T06:00:18+09:00

<p>新しくUSBメモリを購入した．早速，これを使おうと思いPCに挿そうとしたが，思い留まった．「店頭で購入したUSBメモリは本当に安全なのだろうか？」結論から言うと，店頭で購入したUSBメモリは安全ではない．購入したままの状態で使うのは極めて危険である．フォーマットし，暗号化を施してから利用すべきだ. ソフトウェアでできる範囲内で安全性を高める方法について以下の3点にまとめた．</p><ol>	<li>店頭で購入したUSBメモリをウイルススキャンする</li>	<li>店頭で購入したUSBメモリをフォーマットする</li>	<li>店頭で購入したUSBメモリを暗号化して使う</li></ol><!--more--><h3>1. 店頭で購入したUSBメモリをウイルススキャンする</h3><p>ウイルス対策ソフトが最新の状態でウイルススキャンを行うべきだ．なぜなら，過去にUSBメモリの製造過程でウイルスに感染していた事例が存在する．必ずしも店頭で購入したUSBメモリが安全だとは限らない．</p><ul>	<li><a href="http://www3.jvckenwood.com/support/info/usb/index.html">2008年11月28日「販売促進用ストラップ型USBメモリ 不具合のお知らせとお詫び」</a></li>	<li><a href="https://www.justmyshop.com/camp/atokmac2008/owabi.html">2008年07月28日「【お詫びとご報告】「ATOK 2008 for Mac」に同梱のUSBメモリについて」</a></li>	<li><a href="http://buffalo.jp/support_s/ruf2-fs/">2007年10月10日「USBフラッシュメモリ不具合のお知らせとお詫び」</a></li></ul><h3>2. 店頭で購入したUSBメモリをフォーマットをする</h3><p>US DoD 5220-22-M規格<a href="#USDoD5220-22-M">*1</a>のような抹消方法を適用後にフォーマットすべきだ．なぜなら，新規で購入した<strong><span style="color:#800000;">USBメモリに自分が知らないファイル</span></strong>が存在することがある．それらのファイルのチェックサムを配布元のホームページなどで照合し確認するすべがなければ安全性はない．よって，使うべきではない．すぐに抹消すべきである．なお，Mac OS X Lionの場合，US DoD 5220-22-M規格が使えるアプリケーション「ディスクユーティリティ」がある．Winsowsの場合はアプリケーション「<a href="http://eraser.heidi.ie/">Eraser</a>」がある</p><h3>3. 店頭で購入したUSBメモリを暗号化して使う</h3><p><a href="http://www.truecrypt.org/">TrueCrypt</a>のようなソフトウェアでUSBメモリを暗号化すべきだ．以下に理由を示す．</p><ul>	<li>USBメモリ紛失時<a href="#c1">*2</a>?<a href="#c2">*3</a>?<a href="#c3">*4</a>に第三者による情報流出を防ぐため暗号化の必要がある</li>	<li>データの確実な消去後も最大で9.8%の復元率がある．暗号化して防ぐ必要がある<a href="#WEI1">*5</a></li>	<li>USBメモリを物理的に壊すには専門的な知識が必要である<a href="#IOD1">*6</a></li></ul><p>TrueCryptでUSBメモリを暗号化する方法については，以下の記事が詳しい．パスワードは長いほうがいいことは言うまでもない．</p><ul>	<li><a href="http://mag.torumade.nu/?p=7109">USBストレージを安全に利用するため「TrueCrypt」で暗号化しよう！</a></li>	<li><a href="http://haaya.net/3579">TrueCryptを使ってUSBメモリを丸ごと暗号化（その１） - HAAYA</a></li></ul><h3>最後に</h3><p>ソフトウェアでできる範囲内で安全性を高める方法について紹介した．</p><div><ul>	<li>USB紛失時，USB廃棄時に暗号化が解かれない限り，データ流出の可能性はない</li>	<li>あなたがデータを渡した相手のウイルス対策意識が低い場合，データ流出の可能性がある</li></ul></div><p>今回の方法は，あくまでも自分のUSBを自分のみが使う場合に有効である．</p><div></div><hr>*1:<a name="USDoD5220-22-M"></a>16進数の00, FF, そして512バイトの乱数で記憶領域内を上書きすることでデータを抹消する*2:<a name="c1"></a>2012年04月27日 広島大学 <a href="http://www.hiroshima-u.ac.jp/news/show/id/13865/dir_id/0">（お詫び）個人情報を含むUSBメモリの紛失について</a>*3:<a name="c2"></a>2012年06月22日 横須賀市 <a href="http://www.city.yokosuka.kanagawa.jp/3130/20120621.html">市民健診受診者の個人情報等が記録されたUSBメモリの紛失について</a>*4:<a name="c3"></a>2008年08月26日 ITmedia <a href="http://www.itmedia.co.jp/news/articles/0808/26/news031.html">受刑者13万人分の個人情報が流出、英国で</a>*5:<a name="WEI1"></a><a href="http://static.usenix.org/events/fast11/tech/full_papers/Wei.pdf">Reliably Erasing Data From Flash-Based Solid State Drives</a> [PDFファイル]*6:<a name="IOD1"></a>2011年08月11日 Excite <a href="http://www.excite.co.jp/News/bit/E1312796893989.html?_p=1">USBメモリーを安全に捨てるためには？</a>
